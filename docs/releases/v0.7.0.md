# aNEOS v0.7.0 Release Notes

## Overview
The 0.7 stabilization series restored cross-project version alignment and
progressively hardened the detection pipeline through three phases of work:
Phase 1 (infrastructure), Phase 2 (physical indicators + factual ground truth),
Phase 3 (real data wiring + validated accuracy).

## Phase 3 Highlights (March 2026)
- **Ground Truth Validation** – First validated detection accuracy against real
  confirmed artificial objects. Tesla Roadster, Centaur stage (2020 SO), and
  Apollo 12 S-IVB (J002E3) all classified correctly at σ≥5. All natural NEOs
  from JPL SBDB correctly classified as natural. Sensitivity=1.00,
  Specificity=1.00, F1=1.00, ROC-AUC=1.00 at calibrated threshold.
- **Physical Indicators Active** – DiameterAnomalyIndicator, AlbedoAnomalyIndicator,
  SpectralAnomalyIndicator now contribute to the scoring pipeline.
- **Real Data Throughout** – DataFetcher wired into Single NEO Analysis, Batch,
  Enhanced Validation Pipeline, and Interactive Mode. JPL SBDB confirmed working.
- **Honest UI** – System status, alert centre, database status, and spectral
  analysis all use real data sources. Fabricated/simulated data removed.
- **G-001 Resolved** – Unit mismatch (km vs m in population stats) and key-name
  mismatch in the physical anomaly scorer fixed. All 4 pre-existing test failures
  resolved; 54/54 tests pass.

## Phase 2 Highlights (March 2026)
- Physical indicators created and wired into scoring pipeline.
- Ground truth dataset builder replaced stub with JPL SBDB Query API +
  Granvik (2018) synthetic fallback.
- GroundTruthValidator with ROC-AUC, sensitivity/specificity, F1 metrics.
- Dashboard fixed to use DetectionManager instead of hardcoded detector.
- DataFetcher async/sync handling, SBDB element parsing, NEOData model
  compatibility all fixed.

## Phase 1 Highlights
- Version alignment: all packages at `0.7.0`.
- Docker-compose fixed, monitoring import guards added, CI groundwork laid.

## Upgrade Notes
1. Expect version `0.7.0` from all aNEOS packages.
2. See `docs/engineering/maturity_assessment.md` for known limitations.
3. Ground truth validation requires network access to JPL SBDB; offline mode
   uses Granvik (2018) synthetic naturals and hardcoded spacecraft fallback.

## Verification
54 automated tests pass. Ground truth validation: sensitivity=1.00,
specificity=1.00 against 3 confirmed artificials + 20 real JPL natural NEOs.
